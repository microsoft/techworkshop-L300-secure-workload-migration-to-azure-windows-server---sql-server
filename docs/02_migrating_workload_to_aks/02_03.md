# Task 03 - Pushing the Container Image to Azure Container Registry

## Introduction

In this task, you will tag and deploy your container image to Azure Container Registry

## Description

In this task, you will 

- Run a CLI command to create the ACR resource in your Azure Resource Group 

- Tag and Push the container image to ACR  

## Success Criteria

* Successfully create a new ACR resource in your Resource Group
* Successfully Tag and Push the conteiner image to your ACR resource in Azure.

## Solution

<details markdown="block">
<summary>Expand this section to view the solution</summary>

1. You will use the resource group you created in the first exercise, it was named `rg-tailspin`, if you used a different name. please replace it in the az command below
2. Create an ACR resource using the followig CLI command:

    ```bash
    az acr create \
      --resource-group rg-tailspin \
      --name tailspinacr1000 \
      --sku Basic
    ```

3. Login to the ACR resource from your terminal

    ```bash
    az acr login --name tailspinacr1000 
    ```

4. Tag the container image

    ```bash
    docker tag aks-demo:1.0 tailspinacr1000.azurecr.io/aks-demo:1.0
    ```

  You can get the `Login Server` URI from the portal in the ACR resource under `Settings` | `Access Keys`

  ![ACR Login Server](../../Hands-on%20lab/images/acr_login_server.png)

5. Push the container image to ACR

    ```bash
    docker push tailspinacr1000.azurecr.io/aks-demo:1.0
    ```

6. After a minute, you can verify the deployment of the container image to ACR by inspecting the `repositories` tab under `Services` in the Azure Portal for teh ACR resource

  ![ACR Repositories](../../Hands-on%20lab/images/acr_repo.png)

</details>
