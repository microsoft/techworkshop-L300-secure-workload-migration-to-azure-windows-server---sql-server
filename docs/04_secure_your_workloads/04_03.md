# Task 03 - Enable Microsoft Defender for Azure SQL Database

## Introduction

Tailspin Toys is migrating their on-premises SQL Server database to Azure SQL Managed Instance. In this task, you will learn how to monitor and improve the security posture of your database using Microsoft Defender for SQL. You will also apply some of the suggested actions to enhance the security of your database and reduce their attack surface.

## Solution

<details markdown="block">
<summary>Expand this section to view the solution</summary>

1. In the **Search resources, services, and docs** box at the top of the Azure portal, search for **Microsoft Defender for Cloud**, then select the **Microsoft Defender for Cloud** service.

2. Expand **Management** in the left navigation menu and select **Environment settings**.

3. Expand the subscription tree at the bottom of the page, and select the subscription you are using for this exercise.

4. On the **Defender plans** page, locate the **Databases** plan and select **Select types**.

    ![Security Databases](../../Hands-on%20lab/images/Security_Databases.png)

    ![Security Databases Types](../../Hands-on%20lab/images/Security_Databases_Types.png)

5. Toggle the `SQL servers on machines` plan or the `Azure SQL Databases` plan to On, depending on your needs.

    ![Security Databases Toggle](../../Hands-on%20lab/images/Security_database_toggle.png)

6. Select **Continue** and then **Save**.

</details>
